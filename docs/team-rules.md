## 커밋컨벤션

```
- [#이슈번호] docs: 작업 설명
- [#이슈번호] chore: 작업 설명
- [#이슈번호] fix: 작업 설명
- [#이슈번호] merge: 작업 설명
```

e.g. [#2] docs: README 메인 소개 페이지 작성

## 브랜치 전략

```
main
└─ 항상 안정 상태

feature/<이슈번호>-<작업명>-<닉네임>
└─ 작업용 브랜치
```

e.g. feature/#1-init-sso

규칙

```
1) main 에 직접 push 금지
2) 모든 작업은 feature branch 에서
3) main 반영은 PR 통해서만
4) PR 머지 전 최소 1명 리뷰
```

## PR 템플릿

```
## 작업 이슈
- #n

## 변경 내용
-
-

## 확인 포인트
- [ ] 커밋 메시지 규칙 지켰는지
- [ ] main 과 충돌 없는지
```
